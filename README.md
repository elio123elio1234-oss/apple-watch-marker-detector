# QuadDot Marker v3 - זיהוי מנח במרחב לשעונים חכמים

## מה זה?

סמן מינימליסטי (QuadDot) מותאם למסכי שעון חכם (OLED) שמאפשר:
- **זיהוי מרחק** מהמצלמה (בסנטימטרים)
- **זיהוי זווית** ביחס למצלמה (X ו-Y)
- **זיהוי מנח תלת-ממדי** מלא (6DOF)
- עובד עם **מצלמות ברזולוציה נמוכה** (סלפי, לפטופ ישן)
- עובד בזוויות תלולות (כולל תנוחת ECG V5/V6)

## למה לא QR / ArUco?

| | QR Code | ArUco | **QuadDot v3** |
|---|---------|-------|-------------|
| מספר פיצ'רים | מאות ריבועים | עשרות ביטים | **4 עיגולים בלבד** |
| גודל מינימלי של פיצ'ר | ~1-2 פיקסלים | ~3-5 פיקסלים | **30-60 פיקסלים** |
| עמידות ל-bloom | נמוכה | נמוכה | **גבוהה מאוד** |
| עמידות לזווית | ~30° | ~40° | **עד ~80°** |
| טווח זיהוי | קרוב מאוד | קרוב | **20-60 ס"מ** |
| דיוק מרכז | פיקסל | פיקסל | **תת-פיקסל** |

## שיפורים ב-v3

### 1. אנטי-bloom (הילה מהמסך)
- בהירות ברירת מחדל 60% (במקום 100%) — מפחיתה bloom משמעותית
- ניתן לשלוט עם `--brightness` (0-100)
- הסרת bright_mask — HSV לבד מספיק סלקטיבי

### 2. זיהוי בזוויות תלולות
- **Ellipse fitting** במקום circularity check — עיגולים הופכים לאליפסות בזווית, עכשיו מזוהים עד יחס 8:1 (~83°)
- גיאומטריית תבנית מורחבת: side_ratio עד 7:1 (במקום 3:1)
- בדיקת dark surround עדינה יותר (טבעת קטנה, סף נמוך)
- מורפולוגיה עדינה (5x5 במקום 7x7) — שומרת על blobs דקים וצרים

### 3. תמיכה בבהירות נמוכה
- HSV Value minimum הורד ל-30 (במקום 50) — תומך בסמנים ב-40% בהירות
- Saturation minimum 80 — דוחה רקעים בצבע עור/קיר בז'

## איך זה עובד?

### הסמן (Marker)
```
    ┌─────────────────┐
    │                 │
    │   ⬤      ●     │   ⬤ = עיגול גדול (כיוון)
    │                 │   ● = עיגול רגיל
    │                 │
    │   ●      ●     │   רקע שחור (OLED = אפס אור)
    │                 │   צבע ירוק (60% בהירות)
    └─────────────────┘
```

- **רקע שחור** — על מסך OLED הפיקסלים כבויים = אפס bloom
- **4 עיגולים ירוקים** גדולים ובולטים (HSV-selective)
- **עיגול אחד גדול יותר** (40%) לזיהוי כיוון (שמאל-עליון)
- בהירות 60% מפחיתה bloom בלי לפגוע בזיהוי

### אלגוריתם הזיהוי
1. **סינון HSV צבעוני** — מסכת צבע ירוק (Hue 30-90, Sat ≥80, Value ≥30)
2. **Ellipse fitting** — התאמת אליפסה לכל blob, קבלת יחס צלעות עד 8:1
3. **בדיקת dark surround** — טבעת חשוכה סביב כל blob (מסך OLED)
4. **התאמת תבנית** — חיפוש 4 blobs שיוצרים מרובע, אחד גדול
5. **סידור נקודות** — מכפלה וקטורית לקביעת סדר TL/TR/BR/BL
6. **הערכת מנח** — `cv2.solvePnP` (IPPE_SQUARE) → מרחק + זוויות

## שימוש

### שלב 1: יצירת הסמן

```bash
# סמן כללי (400x400, ירוק 60% בהירות)
python generate_marker.py

# סמן מותאם לאפל ווטש 45mm
python generate_marker.py --watch-model 45mm

# בהירות נמוכה יותר (פחות bloom)
python generate_marker.py --brightness 40

# בהירות גבוהה (יותר טווח, יותר bloom)
python generate_marker.py --brightness 80

# צבע ציאן במקום ירוק
python generate_marker.py --color cyan

# עם דף HTML להצגה על כל מסך
python generate_marker.py --html
```

### שלב 2: הצגת הסמן על השעון

**אפשרות א' — תמונה על השעון:**
1. שלח את `marker.png` לאייפון
2. הגדר כתמונת Watch Face (Photo face)
3. או פתח את התמונה באפליקציית התמונות על השעון

**אפשרות ב' — HTML (לבדיקה על כל מסך):**
1. הרץ `python generate_marker.py --html`
2. פתח את `marker.html` בדפדפן על הטלפון/טאבלט
3. לחץ למסך מלא

### שלב 3: הפעלת הזיהוי

```bash
# ברירת מחדל (מצלמה 0, גודל 25mm, ירוק)
python detect_marker.py

# עם גודל סמן מותאם (מדוד על המסך!)
python detect_marker.py --marker-size 23.5

# מצלמה אחרת + מצב דיבאג
python detect_marker.py --camera 1 --debug

# צבע ציאן
python detect_marker.py --color cyan --marker-size 25
```

### פקדים בזמן ריצה
| מקש | פעולה |
|-----|-------|
| `q` / `ESC` | יציאה |
| `d` | הפעלת/כיבוי מצב דיבאג |

| `+` | הגברת החלקה (פחות רעידות) |

## פתרון בעיות

### הסמן לא מזוהה
- בדוק שהצבע מתאים (`--color green` בשני הצדדים)
- נסה מצב דיבאג (`--debug`) לראות אלו blobs זוהו
- קרב את השעון למצלמה (20-30 ס"מ לפחות בהתחלה)
- הורד בהירות אם יש bloom חזק: `--brightness 40` ב-generate

### לא מזהה בזווית תלולה
- v3 תומך עד ~80° — אם עדיין לא מזהה:
  - ודא שרואים חלק מהרקע השחור סביב העיגולים
  - הגדל את הסמן (שעון גדול יותר / מסך קרוב יותר)

### יש false positives
- בדוק שאין אובייקטים ירוקים רוויים ברקע
- שנה צבע: `--color cyan` או `--color magenta`

### המרחק לא מדויק
- מדוד את ה-`marker-size` בפועל על המסך
- כייל את המצלמה אם צריך דיוק גבוה

### ביצועים
- אם ה-FPS נמוך: `--width 640 --height 480`
- הגבר החלקה: `--smooth 0.6`
